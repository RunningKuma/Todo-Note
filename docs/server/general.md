# 服务端整体实现
## 定位
服务端的定位主要是为了云同步，在服务端上存储一份数据作为用户权威数据来源。\
因此主要功能点仅在于数据存储，以及常见的数据处理。

## 架构
### 存储数据
服务端应当按照以下层次存储数据：
1. 用户
	1. 基本信息，参考 [user.d.ts](server\src\models\user.ts)
	2. Todo 数据
		1. 所有 Todo 与其状态
	3. 笔记数据
		1. 所有笔记元信息及其内容
		2. 笔记存放结构（虚拟文件夹）

### 后端模型
models 下应当包含以下模型：

1. **user**
	应当处理以下功能逻辑
	3. 用户数据 数据库维护
	1. 用户登录注册处理
	2. 用户基本信息 CRUD
2. **todo**
	1. todo 数据库的维护
	2. todo 数据 CRUD
3. **note**
	1. note 数据库的维护
	2. note 存放结构数据的维护
	3. note 数据 CRUD

对于每个功能点的具体实现参见其它相关文档

## 路由
后端路由应当按照如下结构分类

1. user.ts | /user
	1. /login | /register
	2. /info
2. todo.ts | /todos
3. note.ts | /notes
